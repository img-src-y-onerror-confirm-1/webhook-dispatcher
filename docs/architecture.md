# Architecture overview

webhook-dispatcher is a TypeScript outbound webhook delivery service that runs as an Express HTTP server (hardened with Helmet and CORS) exposing a versioned `/api/v1` REST API for registering, retrieving, and deleting webhook subscriptions and for dispatching events to all matching subscribers. Its entry point (`src/index.ts`) opens a single ioredis connection and wires it into two core services: a `SubscriptionStore` that persists subscriptions in Redis, and a `RetryQueue` that models the delivery pipeline as a Redis sorted set keyed by each entry's next-retry timestamp. When an event is dispatched, every active subscription for that event type is enqueued and a background poller started by the queue pulls due entries and hands them to the delivery module, which uses undici to POST the payload with a timestamped HMAC-SHA256 `X-Webhook-Signature` header (guarding against replay) under a bounded request timeout. Successful 2xx responses are recorded while failures are re-enqueued with exponential backoff plus jitter until `MAX_RETRIES` is exhausted, at which point the webhook is marked permanently failed. Cross-cutting concerns are handled by small, focused modules — Zod-based request validation middleware, a pino logger, HMAC signing utilities, and environment-driven configuration — keeping the routing, storage, queue, and delivery layers cleanly separated.
